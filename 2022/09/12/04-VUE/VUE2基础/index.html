Vue 基础

初识 vue

vue 的特性

  1. 数据驱动视图
    • vue 会监听数据的变化，从而自动重新渲染页面的结构
    • 数据驱动视图是 单向 的数据绑定
  2. 双向数据绑定
    • 可以辅助开发者在不操作 DOM 的前提下，自动把用户填写的内容同步到数据源中
    • 优点：开发者不需要手动操作 DOM 元素，来获取表单元素最新的值

MVVM

MVVM —— Model、View、ViewModel

是 vue 实现数据驱动视图和双向数据绑定的核心原理。它将每个 HTML 页面都拆分为了这三个部分。

  • Model：表示当前页面渲染时所依赖的数据源

  • View：表示当前页面所渲染的 DOM 结构

  • ViewModel：表示 vue 实例，是 MVVM 的核心（扮演中介的角色，将 数据源 Model 和 页面的结构 View 联系到一起）

    • 当数据源发生变化时，会被 ViewModel 监听到，VM 会根据最新的数据源自动更新页面的结构

    • 当表单元素的值发生变化时，也会被 ViewModel 监听到，VM 会把变化过后最新的值自动同步到 Model 数据源中

组件间样式冲突问题

问题原因

写在 vue 组件中的样式会全局生效,因此很容易就造成多个组件样式冲突的问题

根本原因

  • 单页面应用程序,所有 DOM 结构都是基于唯一的 inex.html 页面进行呈现的
  • 每个组件的样式都会影响整个的 index.html 页面的 DOM 元素
问题解决

为 style 节点添加scoped属性

原理：scoped 属性用来自动为每个组件分配唯一的“自定义属性”,并且为当前组建的 DOM 标签和 style 样式应用这个自定义属性,防止组件的样式冲突问题

新增问题

添加 scoped 属性后,当前组件的样式对其子组件是不生效的,即子组件没有被分配 scoped 为其父组件自动分配的属性.

加了 scoped 的组件标签，都为被自动添加上一个自定义属性 data-v-xxx1

可以控制路由的访问权限。

全局前置守卫

通过路由实例对象的 beforeEach() 方法，即可声明全局前置守卫

  • 在 router/index.js 中添加

    • 每次发生跳转的时候就会自动触发 router.beforeEach(fn) 中fn这个函数

    • router.beforeEach((to, from, next) => {})中

      • to 是将要访问的路由的信息对象
      • from 是将要离开的路由信息对象
      • next 是一个函数，调用 next() 表示放行，允许这次路由导航

    • next 函数的 3 种调用方式

      • 用户拥有后台主页访问权限，直接放行，next()
      • 用户没有后台主页访问权限，强制其跳转到登录页面，next(‘/login’)
      • 用户没有后台主页访问权限，不允许跳转到后台主页，next(false)
