编译出错：’v-model’ directives require no argument.eslintvue/no-v-model-argument

错误3.

Already included file name ‘c:/Users/余萍萍/Desktop/学习/前端/前端基础代码/07 Vue案例/魔方/magic-cube/src/components/Cube.vue’ differs from file name ‘c:/Users/余萍萍/Desktop/学习/前端/前端基础代码/07 Vue案例/魔方/magic-cube/src/components/cube.vue’ only in casing.Vetur(1261)

错误原因：创建文件时，开始创建的文件名是小写开头的，后面又改成大写开头的。缓存的判重逻辑是不区分大小写导致的。在这种情况下，vscode缓存中放的是cube.vue，改了大小写后，vscode试图把Cube.vue文件放进缓存是，判别到重复了，就会提示错误

解决方法：

  1. 导入时。去掉vue的后缀名

    `import Cube from ‘./components/Cube

  2. 将大小写改回原来的

  3. 改成别的文件名

  4. 先改成别的文件名，然后重启Vscode，再改为原来的想改的名字

错误4

报错：Component name “Index” should always be multi-word.

解决：

  1. 方法一：修改文件名称

    如：将 index.js 改为 HomeIndex.js

  2. 方法二（网上方法不起作用）：将vue.config.js文件中的语法检查关闭，添加：lintOnSave: false

  3. 方法三：修改eslint规则

    在 eslintrc.js 文件中，添加

    'vue/multi-word-component-names': ['error', { ignores: ['index', 'Layout'] }]

    ignores数组里面填写需要 忽略组件名称格式检查 的组件名
