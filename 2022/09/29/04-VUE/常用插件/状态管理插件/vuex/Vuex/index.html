概述

Vuex是什么

Vuex 是一个专为 Vue.js 应用程序开发的状态管理模式 + 库。它采用集中式存储管理应用的所有组件的状态，并以相应的规则保证状态以一种可预测的方式发生变化

使用Vuex统一管理数据的好处

  1. 能够在vuex中集中管理共享的数据，易于开发和后期维护
  2. 能够高效地实现组件之间的数据共享，提高开发效率
  3. 存储在vuex中的数据都是响应式的，能够实时保持数据与页面的同步

什么数据适合存储到Vuex中

一般只有组件之间共享的数据才有必要存储到vuex中，对于组件中私有的数据，依旧存储在组件自身的data中即可

Action

用于处理异步操作

在 actions 中，不能直接修改state中的数据，要通过触发mutations的方式间接变更数据

定义Action

actions: {
addAsync(context) { // 没有参数时
// context可以理解为一个mini版的Store，里面包含了可能会用到的方法等
setTimeout(() => {
// 必须通过 context.commit() 触发某个mutation才行
context.commit(‘add’);
}, 1000);
}
},

命名空间

开启命名空间

当有多个这样的js文件时，就可以引起数据冲突，因此需要开启命名空间，由此即使出现函数名重叠，也不会引起冲突

namespaced: true // 开启命名空间

如何在组件中使用store中的数据？

(下面的 dataHandle 是模块名称)

  1. 组件中读取 state 数据（两种方式）

    • this.$store.state.dataHandle.count

    • ~~~js
      computed: {
      …mapState(‘dataHandle’, [‘count’]),
      }

  2. 组件中触发 mutations（两种方式）

    • this.$store.commit(‘dataHandle/add’, 参数)

    • ~~~js
      methods: {
      …mapMutations(‘dataHandle’, [‘sub’])
      }

注意

  • 模块内部的 state 是局部的，只属于模块本身所有。
  • 模块内部的 action、mutation 和 getter 默认是注册在全局命名空间的
  • 如果我们只想让他们在当前的模块中生效，应该怎么办呢？– 通过添加 namespaced: true 的方式使其成为带命名空间的模块。
  • 在模块内通过 rootState 可以拿到根模块中的数据

模块化

分为两种情况：

  1. 默认模块

    此时，state 区分模块，其它的 getters mutations actions 都在全局

  2. 带有命名空间的模块

    此时，所有功能区分模块，封装性和复用性更高

模块化后，state尽量写成一个函数（类似vue3中的data一样）
