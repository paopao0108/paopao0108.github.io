BOM概述

什么是BOM

BOM-Browser Object Model 浏览器对象模型，提供了独立于内容而与浏览器窗口进行交互的对象，核心对象是window

BOM是由一系列相关的对象构成，并且每个对象都提供了很多方法和属性

DOM和BOM的区别

DOM BOM
文档对象模型 浏览器对象模型
把【文档】当作一个【对象】来看 把【浏览器】当作一个【对象】来看
顶级对象是【文档】 顶级对象是【window】
主要学习 操作页面元素 学习 浏览器窗口交互的一些对象
是W3C标准规范 是浏览器厂商在各自的浏览器上定义的，兼容性较差

window对象的常见事件

窗口加载事件load

load

window.onload = function() {}

或

window.addEventListener(‘load’, function() {});

window.onload是窗口加载事件，当文档内容（包括图像、脚本文件、CSS文件等）完全加载完成会触发该事件

注意：

  1. 有了window.onload就可以把JS代码写到页面元素的上方，因为onload是等页面全部加载完毕再去执行处理函数
  2. window.onload传统注册方式只能写一次，若有多个window.onload，以最后一个为准
  3. 若使用window.addEventListener，则没有限制

load与DOMContentLoaded

  • window.addEventListener(‘load’, function (){})

    load会等页面内容全部加载完毕后（包含页面dom元素、图片、flash、css等等），再去处理

  • window.addEventListener(‘DOMContentLoaded’, function (){})

    DOMContentLoaded是DOM加载完毕，不包含图片、flash、css等等，就可以执行，加载速度比load快一些

定时器

window 对象提供了两个非常好用的方法：

  • setTimeout()
  • setInterval()

setTimeout()定时器

window.setTimeout(调用函数，[延迟的毫秒数]);

用于设置一个定时器，在定时器到期后执行调用函数

注意

  • 有三种写法

  • 调用定时器时，可以省略window，直接setTimeout(function(){}, 毫秒数)

  • 延时单位是毫秒，可省略，默认是0毫秒；时间到了才去回调里面的函数

  • 调用函数直接写函数 或者 函数名 均可

  • 页面中可能会有很多定时器，可以给定时器加标识符（名字）

    如：var timer = setTimeout(function, 3000);

停止clearTimeout()

window.clearTimeout(timeId);

  • timeid就是定时器的名称
  • window可以省略
  • 此方法取消了setTimeout的建立的定时器

==JS执行队列==

JS执行机制

JS是单线程，同一个时间只能做一件事；为了解决这个问题，H5提出了Web Worker标准，允许JS创建多个线程，于是Js出现了同步和异步，同步和异步的本质区别就是流水线上各个六成的执行顺序不同

同步

同步任务都在主线程上执行，形成一个执行栈

异步

异步通过回调函数实现

异步的几种类型

一般来说，异步任务又以下三种类型：

  • 普通事件，如：click resize
  • 资源加载，如：load error
  • 定时器，如：setInterval setTimeout

异步任务相关回调函数添加到任务队列中

执行机制

  1. 先执行 执行栈中的同步任务
  2. 异步任务（回调函数）放入任务队列中（异步任务会根据谁先触发 就先执行谁）
  3. 一旦执行栈中的所有同步任务执行完毕，系统就会按次序读取任务队列中的异步任务，于是被读取的异步任务结束等待状态，进入执行栈，开始执行

由于线程不断的重复获得任务、执行任务、再获取任务、再执行，所以这种机制被称为事件循环(even loop)

location对象

window对象提供了一个location属性用于获取或设置窗体的URL，并且可以用于解析URL，因这个属性返回的是一个对象，故称之为location对象

URL

Uniform Resource Locator，即URL 统一资源定位符 是互联网上标准的地址，互联网上每个文件都有一个唯一的URL，它包含的信息指出文件位置以及浏览器如何处理它

URL的一般语法格式为：

protocol://host[:port]/path/[?query]$fragment

如：

http://www.itcast.cn/index.html?name=andy&age=18#link

组成 说明
protocol 通信协议，常用http，ftp，maito等
host 主机（域名） 如：www.itheima.com
port 端口号 可选，省略时使用方案的默认端口 如：http的默认端口号为80
path 路径 有零个或多个‘/’符号隔开的字符串，一般用来表示主机上的一个目录或文件地址
query 参数 以键值对的形式，通过&符号分隔开
fragment 片段 #后面内容 常见于链接 锚点

location对象属性

location对象属性 返回值
location.href 获取或设置整个URL
location.host 返回主机（域名）www.itheima.com
location.port 返回端口，若未写，则返回空字符串
location.pathname 返回路径
location.search 返回参数
location.hash 返回片段 #后面内容，常见于链接 锚点

元素偏移量offset

offset概述

offset偏移量，可使用offset相关属性动态得到元素的位置（偏移）、大小等

  • 获得元素距离带有定位父元素的位置
  • 获得元素自身的大小（宽度高度）
  • 注意：返回的数值都不带单位

常用属性

offset系列属性 作用
elem.offsetParent 返回该元素带有定位的父元素；若父级没有定位，则返回body
elem.offsetTop 返回元素相对带有定位的父元素上方的偏移
elem.offsetLeft 返回元素相对带有定位的父元素左边的偏移
elem.offsetwidth 返回自身 包括padding、边框、内容区的宽度，返回数值不带单位
elem.offsetHeight 返回自身 包括padding、边框、内容区的高度，返回数值不带单位

==offset与style区别==

offset style
可得到任意样式表中的样式值 只能得到行内样式表中的样式值
获得数值是没有单位的 获得的数值是带有单位的字符串
offsetWidth包含 padding+border+width style.width不包含padding和border
offsetWidth等是只读属性，不可赋值更改 style.width是可读可写属性，可赋值

总结：

  • 获取元素大小位置，使用offset
  • 更改元素属性值，使用style

mouseenter与mouseover的区别

  • 当鼠标移动到元素上时会触发mouseenter事件
  • 类似mouseover，但mouseover经过自身盒子和其子盒子都会触发；mouseenter只有经过自身才会触发
  • 即mouseenter不会冒泡
  • 与mouseenter相对应的是的mouseleave，同样不会冒泡

版本1-点击后匀速移动

问题bug：

  • 通过点击触发盒子移动，点击次数越多，盒子移动越快

原因：

  • 不断点击按钮，会触发多个定时器，导致速度越来越快

解决：

  • 定时器每次触发前，清除之前的定时器

版本2-修复多次点击加速

问题：

  • 如何实现缓动效果？

解决：

  • 原理：让元素运动速度有所变化，常见的是让速度慢慢变小，盒子缓慢停下来
  • 即让盒子每次移动地距离变小，速度就慢慢落下来了
  • 核心算法：（目标值-现在的位置)/10 作为每次移动的距离

版本3-利用步长实现缓动

注意：

  • 步长 = (目标值 - 当前位置）/ 10
  • setInterval定时器一般间隔15毫秒

问题：

  • 无法准确的到达目标位置target，总是差一点

解决：

  • 原因：由于计算步长时出现的小数，多次叠加导致
  • 利用向上取整，Math.ceil
  • 此处向上取整和向下取整，结果不一致，不是很理解，向下取整还是无法准确的到达目标位置

版本4-步长取整

注意：

  • 更改了步长，让步长取整

版本5-处理正负步长

注意：

  • 步长可正可负：若步长为正，向上取整，步长为负，向下取整
  • 由于步长可正可负，故不能再使用 元素位置是否大于目标位置 来判断，应该使用 是否等于目标位置 来判断

版本6-添加回调函数

  • 回调函数应该再定时器结束后执行
  • 回调函数可省，即如果if(callback) 为真，才执行，否则没有回调函数

轮播图案例

节流阀

防止轮播图按钮连续点击造成播放速度过快

节流阀目的：当上一个函数动画内容执行完毕，再去执行下一个函数动画，然时间无法连续触发

核心实现思路：利用回调函数，添加一个变量来控制，锁住函数和解锁函数

步骤：

  • 开始设置一个变量 var flag = true
  • if (flag) {flag = false; do something} 关闭水龙头
  • 利用回调函数，在动画执行完毕后，flag = true 打开水龙头
