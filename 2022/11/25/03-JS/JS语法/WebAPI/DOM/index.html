DOM

DOM简介

DOM即 Document Object Model 文档对象模型

DOM树

  • 文档：一个页面就是一个文档，DOM中使用 document 表示
  • 元素：页面中所有标签都是元素，DOM中使用 element 表示
  • 节点：网页中所有内容都是节点（标签、属性、文本、注释、换行等），DOM使用 node表示

DOM将以上内容都看作对象

如何获取元素

  • 根据ID获取
  • 根据标签名获取
  • 通过HTML5新增的方法获取
  • 特殊元素获取

id获取元素

var element = document.getElementById(id);

  • element是一个Element对象，
  • id是大小写敏感的字符串，代表了要查找元素的唯一ID
  • 返回值是 一个匹配到ID的DOM Element对象，若在当前Document下没有找到，则返回null

说明：

  • 由于文档是从上往下加载，因此需要先写标签，再写script
  • 参数id是大小写敏感的字符串
  • 返回的是一个元素对象
  • console.dir(元素对象) 打印返回元素对象、可查看里面的属性和方法

标签名获取元素

使用getElementsByTagName()方法返回带有标签名的对象的集合

var element = document.getElementByTagName(‘标签名’);

注意:

  • 由于通过标签名获取元素是一个对象的集合,所以要想操作里面的元素就需要遍历
  • 得到的元素是动态的
  • 返回值是 获取的对象的集合，并以伪数组的形式存储

HTML5方法获取元素

  1. 根据类名返回元素对象集合

    var element = document.getElementByClassName(‘类名’);

  2. 根据指定选择器返回第一个元素对象

    var element = document.querySelector(‘选择器’);

  3. 根据指定选择器返回对象集合

    var element = document.querySelectorAll(‘选择器’);

注意:

  • querySelector 返回指定选择器的第一个元素对象， 里面的选择器需要加符号

    若是id选择器,则需要在id名前加 # ;

    若是类选择器,则需要在类名前加 . ;

  • querySelectorAll 返回指定选择器的所有元素对象

特殊获取元素

获取body元素

返回body对象

document.body;

获取html元素

返回html

document.documentElement;

案例-全选反选

分析：两种情况

  • 点击第一个选框：第一个选中，下面全选；第一个不选，下面全不选
    • 具体思想：第一个选框的状态决定了下面所有选框的状态，因此当鼠标点击第一个选框的时候，将下面所有选框的属性值赋予第一个选框相同的值即可
  • 点击下面的选框：只要下面全选，第一个才选；下面有一个不选，第一个也不选
    • 具体思想：选中下面任意一个选框时，都需要判断此时下面所有选框是否被选中，若有一个不被选中，则第一个选框不选。可设置一个flag，每次点击下面的选框，flag均置为1，再循环判断是否有不被选中的，若是则flag置为0，跳出判断；将第一个选框的值置为flag

节点操作

为什么利用节点操作元素？

获取元素的方式通常有两种：

  1. 利用DOM提供的方法获取元素

    document.getElementById(id)

    document.getElementByTagName(‘标签名’)

    document.getElementByClassName(‘类名’)

    document.querySelector(‘选择器’)

    等等

    缺点：逻辑性不强，且繁琐

  2. 利用节点层级关系获取元素

  • 利用父子兄节点关系获取元素
  • 逻辑性强，兼容性差

节点概述

网页中所有内容都是节点（标签、属性、文本、注释等），在DOM中，节点使用node来表示

节点属性

节点至少拥有三个属性：

  • nodeType（节点类型）
  • nodeName（节点名称）
  • nodeValue(节点值)

==节点类型==

  • 元素节点 nodeType 为 1
  • 属性节点 nodeType 为 2
  • 文本节点 nodeType 为 3 (文本节点包含文字、空格、换行等)

实际开发中主要操作元素节点

节点层级

常见 父子兄层级关系

父级节点

node.parentNode

  • parentNode属性可以返回某节点的父节点，注意是最近的一个父节点
  • 若指定节点没有父节点，则返回null

子节点

  1. parenNode.firstChild
  • 返回第一个子节点，若不存在则返回null（不管是文本节点，如换行、空格都算，还是元素节点）

  • 返回最后一个子节点，若不存在则返回null（不管是文本节点，如换行、空格都算，还是元素节点）

3和4有兼容性问题：IE9以上支持

DOM事件高级

注册事件

传统方式

  • 利用on开头的事件，如onclick

  • 写法：

    • <button onclick=”alert(‘hi~’)”></button>
    • btn.onclick = function() {}

  • 特点：注册事件的唯一性

    如：若同一个注册事件添加了两个处理程序，则后面的会把前面的覆盖，只会执行最下面那个

    • btn.onclick = function() {alert(‘hi~’)}
    • btn.onclick = function() {alert(‘你好’)} // 只会执行这个

  • 同一个元素同一个事件只能设置一个处理函数，最后注册的处理函数将会覆盖前面注册的处理函数

方法监听注册方式

  • 方法：addEvenListener
  • IE9之前不支持addEvenListener方法，可以使用attachEvent()
  • 特点：同一根元素同一个事件可以注册多个监听器
  • 按注册顺序依次执行

addEventListener事件监听方式

evenTarget.addEventListener(Type, listener[, useCapture])

evenTarget.addEvenListener()将指定的监听器注册到evenTarget(目标对象)上，当该对象触发指定事件时，就会执行事件处理函数

IE9之前不支持addEvenListener方法

  • type：事件类型字符串，如：click、mouseover（注意：这里不需要带 on）
  • listener：事件处理函数，事件发生时，会调用该监听函数
  • useCapture：可选参数，是一个布尔值，默认是false

attachEvent事件监听方式

evenTarget.attachEvent(evenNameWithOn, callback)

evenTarget.attachEvent()将指定监听器注册到evenTarget(目标对象)上，当该对象触发指定事件时，指定的回调函数就会被执行

  • evenNameWithOn：事件类型字符串，如：click、onmouseover，（注意：这里要带 on）
  • callback：事件处理函数，当目标触发事件时回调函数被调用

==DOM事件流-捕获、冒泡==

事件流描述的是从页面中接收事件的顺序

事件发生时会在元素节点之间按照特定的顺序传播，这个传播过程即DOM事件流

DOM事件流分为三个阶段

  1. 捕获阶段 (如：document $\longrightarrow$ html $\longrightarrow$ body $\longrightarrow$ father $\longrightarrow$ son)
  2. 当前目标阶段
  3. 冒泡阶段 （如：son $\longrightarrow$ father $\longrightarrow$ body $\longrightarrow$ html $\longrightarrow$ document)

注：

  • JS代码中只能执行 捕获 或者 冒泡 其中的一个阶段

  • onclick和attachEvent 只能得到冒泡阶段, addEventListener可以得到捕获和冒泡阶段

  • 捕获阶段：addEventListener的第3个参数是true，那么处于捕获阶段

    如：document $\longrightarrow$ html $\longrightarrow$ body $\longrightarrow$ father $\longrightarrow$ son

  • 冒泡阶段：addEventListener的第3个参数是 省略 或者 false，那么处于冒泡阶段

    如：son $\longrightarrow$ father $\longrightarrow$ body $\longrightarrow$ html $\longrightarrow$ document

  • 有些事件没有冒泡，如：onblur onfocus onmouseenter onmouseleave

事件对象

什么是事件对象

解释：event对象海边时间的状态，比如：键盘按键的状态，鼠标的位置，鼠标按钮的状态

即：事件发生后，跟事件相关的一系列信息数据的集合都放到这个对象里面，这个对象就是事件对象event，有很多属性和方法

具体来说：

  • event就是一个事件对象，写在监听函数小括号里面，当作形参来看
  • 事件对象只有有了事件才会存在（如，需要点击之后才会产生），是系统自动创建的，不需要我们传递参数
  • 事件对象是 一系列相关数据的集合，跟事件相关的，如鼠标点击里面就包含了鼠标的相关信息
  • 事件对象可以看成形参自定义命名，如：event、e、evt
  • 事件对象也有兼容性问题，ie6 7 8 通过 window.event

事件对象属性方法

事件对象中常见的属性和方法如下：

事件对象属性方法 说明
e.target 标准 返回触发事件的对象 标准
e.srcElement 非标准 返回触发事件的对象 非标准 ie6-8使用
e.type 返回事件的类型 如click，不带on
e.cancelBubble 非标准 该属性阻止冒泡 非标准 ie6-8使用
e.returnValue 非标准 该属性阻止默认事件（默认行为）非标准 ie6-8使用 如：不让链接跳转
e.preventDefault 标准 该属性阻止默认事件（默认行为）标准 如：不让链接跳转
e.stopPropagation 标准 阻止冒泡

阻止默认行为

如：不让链接跳转，不让提交按钮提交

  • e.returnValue; 是一个属性，低版本浏览器

  • e.preventDefault(); 是一种方法，调用需要加括号，标准普通浏览器

  • return false; 也可以阻止默认行为，但后面代码不执行了，只限于传统的注册方式（e.onclick）没有兼容性问题

阻止事件冒泡

  • e.stopPropagation()
  • e.cancelBubble = true 非标准阻止冒泡（低版本使用）

兼容性解决方案

事件委托

事件委托 也称为 事件代理，在jQuery里面称为事件委派

事件委托原理：

不用每个子节点单独设置事件监听器，而是事件监听器设置在其父节点上，然后利用冒泡原理影响每个子节点

事件委托作用：

只操作了一次DOM，提高了程序的性能

键盘事件

键盘事件 触发条件
onkeyup/keyup 某个键盘按键被松开时触发
onkeydown/keydown 某个键盘按键被按下时触发
onkeypress/keypress 某个键盘按键被按下时触发 不识别功能键，如：ctrl shift 箭头 等

三个事件的执行顺序：

keydown $\longrightarrow$ keypress $\longrightarrow$ keyup

  • addEventListener 不需要加 on

keyCode

键盘事件中的keyCode属性可以得到相应键的ASCII码值

  • keyup和keydown不区分大小写（a A 都是A的ASCII：65），能识别任何键
  • keypress 区分大小写，不能识别功能键
