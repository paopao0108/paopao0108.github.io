监听GET请求

通过app.get()方法，可以监听客户端的GET请求，（在上面的第2步到第3步之间，即创建web服务器后，启动服务器前）

  • get第一个参数：客户端请求的URL地址
  • get第二个参数：请求对应的处理函数
  • 函数参数req：请求对象（包含了与请求相关的属性和方法）
  • 函数参数res：响应对象（包含了与响应有关的属性和方法）

托管静态资源

express.static()
  • 通过express.static()可以非常方便的创建一个静态资源服务器

案例：将之前clock文件夹里面的文件通过express.static() 对外开放访问

clock文件夹里面有三个文件index.html, index.css, index,js，通过上面的操作，在网页上只需要输入 http://127.0.0.1后面加上 /index.html 或 /index.css 或 /index.js 即可访问到对应文件，不需要写 /clock/index.html

注意：

  • express在指定的静态目录中查找文件，并对外提供资源的访问路径，存放静态文件的目录名不会出现在URL中，如上所示，访问时的url地址不需要写 /clock

  • 若要托管多个静态资源目录，可以多次调用express.static()函数

    访问静态资源时，会根据目录的添加书匈奴查找所需的文件，即 访问的资源，若能够在public找到，就用public的，若public没有才去下面的查找

  • 挂载路径前缀

    此时就需要有/clock前缀才能访问到指定的文件，如：http://127.0.0.1/clock/index.html

nodemon

在编写调试Node.js项目，使用node启动项目时，若修改了项目的代码，需要频繁手动的close，然后再重新启动，很繁琐。

nodemon能够监听项目文件的变化，当代码被修改后，nodemon会自动帮我们重启项目

安装

npm install -g nodemon

使用

只需要将原来的 node app.js 的node 改为 nodemon 即可

nodemone app.js

Express中间件

中间件：业务流程的中间处理环节

中间件基础

中间件的格式

本质上就是一个function处理函数，在中间件的形参列表中，必须包含next参数。而路由处理函数中只包含req和res

next函数的作用

next() 是实现多个中间件连续调用的关键，表示吧流转关系转交给下一个中间件 或 路由

定义中间件函数
  • 全局生效的中间件
    • 客户端发起的任何请求，到达服务器后，都会触发的中间件，叫做全局生效的中间件
    • 通过调用app.use(中间件函数)，即可定义一个全局生效的中间件

定义中间件的简化形式

中间件的作用

在中间件函数里面定义了time，并挂载到乐req的自定义属性里面，由此就不必在每个路由中都重新定义一个时间time

注意事项
  • 一定要在路由之前注册中间件
  • 执行完中间件的业务员代码后，一定要调用next()函数
  • 调用next()后 不要再写额外的代码
  • 连续调用多个中间件时，多个中间件之间，共享 req 和 res 对象

中间件的分类

  • 应用级别的中间件

    通过app.use(), app.get(), app.post(), 绑定到app实例上的中间件

  • 路由级别的中间件

    绑定到express.Router()实例上

  • 错误级别的中间件

    错误级别的中间件的function处理函数中，必须有4个参数，参数的顺序从前到后，分别是（err, req, res, next)

    特别注意的是：错误级别的中间件必须注册在所有路由之后，这是和之前不一样的

  • Express内置的中间件

    • express.static

      快速托管静态资源的内置中间件（无兼容性）

    • express.json

      解析JSON格式的请求体数据（有兼容性，仅在4.16.0以后的版本可用）

      ![image-20220529110437125](https://oss-pic-bucket.oss-cn-shanghai.aliyuncs.com/img/06 前后端交互/image-20220529110437125.png)

    • express.urlencoded

      解析URL-encoded格式的请求体数据（有兼容性，仅在4.16.0以后的版本可用）

      ![image-20220529111433938](https://oss-pic-bucket.oss-cn-shanghai.aliyuncs.com/img/06 前后端交互/image-20220529111433938.png)

  • 第三方的中间件

    由第三方开发出来的中间件，第三方中间件可以按需下载并配置

    常用body-parser这个第三方中间件来解析请求体数据。

    • 运行 npm install body-parser 安装中间件
    • 使用 require 导入中间件
    • 调用 app.use() 注册并使用中间件

    express内置的express.urlencoded中间件，就是基于body-parser这个第三方中间件进一步封装出来的
