fs文件系统模块

fs文件系统模块

fs模块是Node.js官方提供的、用来操作文件的模块，提供了一系列的方式和属性，用来满足用户对文件的操作需求。如：

  • fs.readFile() 用来读取指定文件中的内容
  • fs.writeFile() 用来向指定的文件中写入内容

使用fs模块来操作文件，需要先导入：

const fs = require(‘fs’)

读取文件内容

fs.readFile

fs.readFile(path[, options], callback)

  • 参数1：必选参数，字符串，表示文件的路径
  • 参数2：可选参数，表示以什么编码格式来读取文件，一般默认指定 utf8
  • 参数3：必选参数，文件读取完成后，通过回调函数拿到读取的结果，回调函数的参数有两个 err 和 datastr

读取文件成功时：

  • 第一个参数 为 null
  • 第二个参数 为 读取的内容

读取文件失败时：

  • 第一个参数 为 错误对象
  • 第二个参数 为 undefined

向文件写入内容

fs.writeFile

fs.writeFile(file, data[, options], callback)

  • 参数1：必选参数，需要制定一个文件路径的字符串，表示文件的存放路径
  • 参数2：必选参数，表示要写入的内容
  • 参数3：可选参数，表示已什么格式写入文件内容，默认值是utf8
  • 参数4：必选参数，文件写入完成后的回调函数，回调函数的参数有err

若写入成功，err为 null；若失败，err为一个错误对象

路径动态拼接问题

问题

使用f模块操作文件时，若提供的操作时以 ./ 或 ../ 开头的相对路径时，容易出现路径动态拼接错误的问题。

原因

代码在运行时，会以执行node命令时所处的目录，动态拼接出 被操作文件 的完成路径

![image-20220525105935028](https://oss-pic-bucket.oss-cn-shanghai.aliyuncs.com/img/06 前后端交互/image-20220525105935028.png)

![image-20220525105143366](https://oss-pic-bucket.oss-cn-shanghai.aliyuncs.com/img/06 前后端交互/image-20220525105143366.png)

如上所示，执行文件06在Day1目录下，若不在Day1目录下执行该文件，文件路径会被动态的拼接为 当前目录地址 + 使用fs.readFile里面提供的地址

![image-20220525110448104](https://oss-pic-bucket.oss-cn-shanghai.aliyuncs.com/img/06 前后端交互/image-20220525110448104.png)

如上所示，由此读取文件就失败了

解决

可以在为fs.readFile里面提供地址时，提供一个完整的文件路径，即绝对路径，就可以解决这个问题

![image-20220525111649159](https://oss-pic-bucket.oss-cn-shanghai.aliyuncs.com/img/06 前后端交互/image-20220525111649159.png)

  • 注意：路径粘贴后，将所有的 \ 改为 \\

  • 问题：移植性差，不利于维护

  • 改进：利用__dirname来拼接目录路径

    __dirname表示当前文件所处目录

  • 注意：使用字符串拼接时，__dirname + ‘/files/1.txt’，第一个/前面不能再加 . ,否则路径拼接出来仍然无法读取。
  • 后续尽量避免使用 + 来拼接路径，容易出错，后续使用path模块来处理路径
