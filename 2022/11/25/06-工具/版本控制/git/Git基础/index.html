Git基础

Git基础

什么是Git？

  • Git是一个开源的分布式版本控制系统，是目前世界上最先进、最流行的版本控制系统。可以快速高效的处理从很小到非常大的项目版本管理
  • Git快照是在原有文件版本的基础上重新生成一份新的文件，类似于备份。为了效率，若文件没有修改，Git不再重新存储该文件，而是保存一个链接指向之前存储的文件
  • 在Git中绝大多数的操作都是只需要访问本地文件和资源，一般不需要来自网络上其他计算机的信息
    • 断网后依旧可以在本地对项目进行版本管理
    • 联网后，把本地修改的记录同步到云端服务器即可

传统的版本控制系统是基于差异的版本控制，存储的是一组基本文件和每个文件随事件逐步累积的差异（例如：SVN）

Git的三个区域

  • 工作区
  • 暂存区：已完成的工作的临时存放区域，等待被提交
  • Git仓库区：最终的存放区域

Git的三种状态

  • 已修改modified

    即 工作取得文件被修改了，但还没有放到暂存区

  • 已暂存staged

    即 若文件已修改并放入暂存区

  • 已提交committed

    即 若Git仓库中保存着特定版本的文件

Git的基本操作

获取Git仓库的方式

  • 将尚未进行版本控制的本地目录转换为Git仓库
  • 从其他服务其克隆一个已存在的Git仓库

工作区文件的4种状态

  • Untracked 未跟踪（只有这种状态是未被Git管理，以下三种状态都是 已被Git管理）

  • Unmodified 未修改

    工作区内容和Git仓库中文件内容一致

  • Modified 已修改

    工作区文件内容和Git仓库中文件内容不一致

  • Staged 已暂存

    工作区中被修改的文件已被放到暂存区，准备将修改后的文件保存到Git仓库中

Git操作的最终结果：让工作区的文件都处于“未修改”的状态

Git关键字解释

HEAD 当前版本的指针，当切换本地版本的时候会快速指向指定版本文件

master git为我们创建主分支

origin 远程仓库的名称

:star2:基本操作及命令

1 初始化仓库

git init

2 检查文件的状态

# 详细的方式显示文件状态

git status

# 以精简的方式显示文件状态

git status -s
git status –short

-s 是 –short 的缩写

3 跟踪新文件

使用 git add 开始跟踪一个文件，所以要跟踪index.html文件，运行如下命令：git add index.html

git add index.html

4 提交更新

现在暂存区有一个index.html文件等待被提交到Git仓库中进行保存，可以执行git commit 命令进行提交，其中-m选项后面是本次提交消息，用来对提交内容做进一步的描述

git commit -m “新建了index.html文件”

提交成功后，可以用git status 再次检查文件的状态，若没有任何文件需要被提交，则证明工作区中所有的文件都处于“未修改”的状态。

5 对已提交的文件进行修改

目前index.html文件已经被Git跟踪，并且工作区和Git仓库中的index.html文件内容保持一致。当我们修改工作区的中的index.html的内容后，会出现changes not staged for commit 这行下面，说明已跟踪文件的内容发生了变化，但还没有放到暂存区。

6 暂存已修改的文件

index.html被修改后，若要暂存这次修改，需要再次运行 git add 命令，这条命令主要有3个功能：

  • 跟踪新文件
  • 把 已跟踪、且已修改 的文件放到暂存区
  • 把有冲突的文件标记为 已解决状态
7 提交已暂存的文件

再次运行 git commit -m “提交消息”

git commit -m “修改了index.html的内容”

8 撤销对文件的修改

即 将工作区中对应文件的修改 还原为 Git仓库中所保存的版本。

操作的结果：所有的修改会丢失，且无法恢复，危险性比较高

撤销操作的本质：用Git仓库中保存的文件，覆盖工作区中指定的文件

9 向暂存区一次性添加多个文件

若需要被暂存的文件个数比较多，可以使用如下的命令，一次性将所有的新增和修改过的文件加入暂存区

git add .

10 取消暂存文件

若需要从暂存区中移除对应的文件，可使用：

git reset HEAD 要移除的文件

11 跳过使用暂存区域

Git标准工作流程是： 工作区 → 暂存区 → Git仓库

为简化工作流程，可以指直接将工作区中修改的提交到Git仓库，这时候Git工作的流程简化为： 工作区 → Git仓库，跳过了 git add 步骤

git commit -a -m “提交消息”

12 移除文件

从Git中移除文件的方式有两种：

  • 从Git仓库和工作区中同时移除对应的文件

    git rm -f index.js

  • 只从Git仓库中移除指定的文件，但保留工作区中对应的文件

    git rm –cached index.css

13 忽略文件

有些文件无需纳入Git的管理，不希望他们出现在未跟踪的文件列表。

这种情况下，可以创建一个名为 .gitignore 的配置文件，列出要忽略的文件的匹配模式

文件 .gitignore 的格式如下：

  • 以 # 开头的是注释
  • 以 / 结尾的是目录
  • 以 / 开头防止递归

glob模式

即简化了的正则表达式：

  • 星号* 匹配零个或多个任意字符
  • [abc] 匹配任何一个列在方括号中的字符
  • 问号？ 只匹配一个任意字符
  • [0-9] 短划线分割的两个字符，表示所有在这两个字符范围内的都可以匹配
  • 两个星号** 表示匹配任意中间目录 （如： a/**/z 可以匹配 a/z a/b/z a/b/c/z）

14 查看提交历史

  • 按时间先后顺序列出所有提交历史,最近提交排在最上面

    git log

  • 只展示最新的两条提交历史,数字可以按需进行填写

    git log -2

  • 在一行上展示最近提交历史的信息

    git log -2 –pretty=online

  • 在一行上展示最近两条提交历史的信息,并自定义输出的格式

    %h 提交的简写哈希值,%an作者名字, %ar作者修订日期,按多久以前的方式显示, %s提交说明

    git log -2 –pretty=format:”%h | %an | %ar | %s”

    其中 | 为自定义的分隔符,可以换成别的字符

15 回退到指定的版本

  • 使用 git reset –hard 命令,根据指定的提交 ID 回退到指定版本

    git reset –hard

    根据git log 查看提交历史,可以查看到版本的ID,再利用上面的命令即可回退到指定版本

  • 在旧版本中使用 git reflog –pretty=oneline 命令,查看命令操作的历史

    git reflog –pretty=oneline

    如果切换到了旧版本,只能用上面这条命令来查看所有的操作历史

  • 再次根据最新的提交 ID, 跳转到最新的版本

    git reset –hard
