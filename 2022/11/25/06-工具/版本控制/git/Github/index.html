开源项目托管平台

专门用于免费存放开源项目源代码的网站,角坐开源项目托管平台.比较出名的主要有以下3个:

  • Github(全球最牛的开源项目托管平台)
  • Gitlab(对代码私有性支持较好,企业用户较多)
  • Gitee(又叫做 码云, 是国产的开源项目托管平台,访问速度快,纯中文界面,使用友好)

注意:由于以上3个平台,只能托管以Git管理的项目源代码,因此,他们的名字都以Git开头

2 远程仓库的访问方式

  • HTTPS

    零配置,但是每次访问仓库是,需要重复输入Github的账号和密码才能访问成功

  • SSH

    需要进行额外的配置,但是配置成功后,每次访问仓库,不需要重复输入Github的账号和密码

实际开发中,更推荐事件SSH的方式访问远程仓库

2.a 基于HTTPS将本地仓库上传到Github

  • 若本地没有现成的Git仓库

    1. 使用终端命令创建README.md文档,并写入初始内容为 # project_01

      echo “# project_01” >> README.md

    2. 初始化本地Git仓库,并将文件的修改提交到本地的Git仓库

      git init

      git add README.md

      git commit -m “first_commit”

    3. 将本地仓库和远程仓库进行关联,并把远程仓库命名为origin

      git remote add origin “url地址”

    4. 将本地仓库中的内容推送到远程origin仓库中

      git push -u origin master

  • 若本地有现成的Git仓库

    1. 将本地仓库和远程仓库进行关联,并把远程仓库命名为origin

      git remote add origin “url地址”

    2. 将本地仓库中的内同推送到远程origin仓库中

      git push -u origin master

2.b 基于SSH将本地仓库上传到Github

SSH key

作用：实现本地仓库和Github之间的免登录的加密数据传输

好处：免登录身份认证、数据加密传输

由两部分组成：

  • id_rsa（私钥文件，存放于客户端的电脑即可）
  • id_rsa.pub（公钥文件，需要配置到Github中）
生成SSH key

  • 打开Git Bash

  • 粘贴如下命令

    ssh-keygen -t rsa -b 4096 -C “注册Github账号时填写的邮箱”

  • 连续敲击3次回车，即可在C:\users\用户名文件夹\.ssh目录中生成 id_rsa 和 id_rsa.pub 两个文件

配置SSH key
  • 用记事本打开 id_rsa.pub 文件，复制里面的文本内容
  • 在浏览器中登录Github，点击头像 → Setting → SSH and GPG Keys → New SSH key
  • 将 id_rsa.pub 文件的内容，粘贴到key对应得文本框中
  • 在Title文本框中人已填写一个名称，来标识这个key从何而来
检测SSH key是否配置成功

打开Git Bash 输入以下命令并回车执行

ssh -T git@github.com

在提示消息后面输入 yes，收到连接成功得提示消息，证明SSH key 就配置成功了

本地分支操作

为防止互相干扰，提高协同开发得体验，建议每个开发者都基于分支进行项目功能的开发

master分支

在初始化Git仓库时，Git默认已经创建了一个名字叫做master的分支，通常把这个master分支叫做主分支

在实际工作中，master主分支的作用是：用来保存和记录整个项目已完成的功能代码

因此不允许程序员直接在master分支上修改代码

功能分支

专门用来开发新功能的分支，临时从master主分支上分叉出来的，当新功能开发且测试完毕后，最终需要合并到master分支上

:star2:命令行操作

1 查看分支列表

git branch

运行结果中，分支前面有 * 号 ，表示当前所处的分支

2 创建新分支

基于当前分支，创建一个新的分支

git branch 分支名称

3 切换分支

可切换到指定的分支上进行开发

git checkout 分支名称

4 分支的快捷创建和切换

git checkout -b 分支名称

-b 代表创建一个新的分支

checkout表示切换到刚才新建的分支上

相当于是 创建分支和切换分支两条命令的简写形式

5 合并分支

切换到主分支master上

git checkout master

在master分支上运行 git merge 命令，将 分支的代码合并到 master 分支

git merge 分支名称

6 删除分支

注意：待删除分支不能是当前所处分支

git branch -d 分支名称

当遇到无法删除时，可以强制删除

git branch -D 分支名称

7 遇到冲突时的分支合并

在两个不同的分支中，对同一个文件进行不同的修改，Git就没办法合并他们，此时需要打开这些包含了冲突的文件，然后手动解决冲突

手动解决完冲突后，再执行以下命令

git add .

git commit -m “解决了分支合并冲突的问题”

:star2:远程分支操作

1 将本地分支推送到远程仓库

若是第一次推送分支需要 带-u 的参数，此后可以直接使用 git push 推送到 远程分支

git push -u 远程仓库的别名 本地分支名称：远程分支名称

如：

git push -u origin payment:pay

若希望远程分支名称和本地分支名称保持一致，可以不写冒号后面的重命名

git push -u origin payment

2 跟踪分支

即从远程仓库中，把远程分支下载到本地仓库中

git checkout 远程分支名称

如：

git checkout pay

从远程仓库中，把对应的远程分支下载到本地仓库，并把下载的本地仓库分支进行重命名

git checkout -b 本地分支名称 远程仓库名称/远程分支名称

如：

git checkout -b payment origin/pay

3 拉取远程分支的最新代码

在远程分支编辑的代码，可以下载到本地对应的分支中

git pull

4 删除远程分支

git push 远程仓库名称 –delete 远程分支名称

如：

git push origin –delete pay
