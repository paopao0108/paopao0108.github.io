核心概念

State

提供唯一的公共数据源，所有共享的数据都要统一放到Store的State中进行存储、

创建State数据

访问State数据

组件访问State中数据方式：

方式1 this.$store.state.xxx

this.$store.state.全局数据名称

模板中使用 不需要写this

方式2 mapState

映射到 computed 中

import { mapState } from ‘vuex’

computed: {

​ // 将全局数据映射为当前组建的计算属性

​ . . .mapState([‘count’])

}

Mutations

用于变更Store中的数据

  • 只能通过mutation变更Store数据，不可以直接操作Store中的数据
  • 通过这种方式虽然操作起来稍微繁琐，但是可以集中监控所有数据的变化

定义Mutations

在store/index.js文件中定义mutations

注意：传递参数 & 没传参数

触发Mutations

方式1 this.$store.commit(xxx)

没有传参时：this.$store.commit(‘mutations里面的函数名’)

传递参数时：this.$store.commit(‘mutations里面的函数名’, 参数)

方式2 mapMutations

映射到 methods 方法中

// 1. 从vuex中导入 mapMutations 函数

import { mapMutations } from ‘vuex’

methods: {

​ // 2. 将指定的 mutations 函数映射为当前组建的 methods 方法

​ . . .mapMutations([‘add’])

}

注意

不要在 mutations 函数中执行异步操作

如：

要处理异步操作，应该在actions中

Actions

用于处理异步操作

在 actions 中，不能直接修改state中的数据，要通过触发mutations的方式间接变更数据

定义Actions

actions: {
addAsync(context) { // 没有参数时
// context可以理解为一个mini版的Store，里面包含了可能会用到的方法等
setTimeout(() => {
// 必须通过 context.commit() 触发某个mutation才行
context.commit(‘add’);
}, 1000);
}
},

触发Action

方式1 this.$store.dispatch(xxx)

这里的 dispatch 函数，专门用来触发 action
没传参数时：this.$store.dispatch(‘actions里面函数名’);

传递参数时：this.$store.dispatch(‘actions里面的函数名’, 参数);

方式2 mapActions

映射到 methods 方法中

// 1. 从vuex中导入 mapActions 函数

import { mapActions } from ‘vuex’

methods: {

​ // 2. 将指定的 actions 函数映射为当前组建的 methods 方法

​ . . .mapActions([‘addAsync’, ‘addNAsync’])

}

Getters

Getter用于对Store中的数据进行加工处理形成的新的数据

  • Getter可以对Store中已有的数据加工处理之后形成新的数据，类似 Vue 的计算属性
  • Store中的数据发生变化，Getter里面的数据也会跟着变化
