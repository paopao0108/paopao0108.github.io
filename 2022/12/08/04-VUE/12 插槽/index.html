插槽

允许开发者在封装组件时，把不确定的、希望由用户指定的部分定义为插槽。简单来说可以认为是：组件封装期间，为用户预留的内容的占位符

基础用法

  • 如果Left组件中没有slot来预留内容占位符，那么 在App组件中的p标签里面的内容就会被丢弃，而无法显示

  • Left组件中slot标签里面的内容会在 Left组件的使用者(App组件)没有提供内容时 显示。如下

作用域插槽

数据在组件的自身，但根据数据生成的结构需要组件的使用者来决定

<slot :user=”userinfo” :msg=”msg”></slot>

注意

  • 若没有解构传过来的数据，即 #default=”user, msg”，则的内容是：

    { “user”: { “name”: “zs”, “age”: 18 }, “msg”: “这是一段文字” }

  • 当通过解构 #default=”{ user, msg }” 后，就可以直接拿到user和msg里面的内容，则的内容是：{ {“name”: “zs”, “age”: 18 }
