VUE指令

内容渲染指令

  1. v-text
    • 会覆盖元素内默认的值
    • 不存在闪动
    • 只能渲染纯文本内容
  2. {{}}
    • 插值表达式，专用于解决v-text会覆盖默认文本内容的问题
    • 存在闪动问题
    • 只能渲染纯文本内容
  3. v-html
    • 会覆盖元素内默认的值
    • 存在闪动问题
    • 可以解析HTML

补充：

  • v-pre

    显示原始信息，跳过编译过程（直接将内容显示出来，不编译）

事件参数对象

$event

  • 当使用add方法时，不传参数时，这里的e就是事件event，就有事件的一系列属性和方法
  • 但当使用add方法，并向add传递参数，如 add(1),那么event就会被覆盖掉，就没有事件的方法和属性

问题：因此如何做到传参和访问事件？

由于vue提供了内置变量，$event就是原生DOM的事件对象，通过向函数传递$event, 就可以在函数内部获取事件

事件注册时：@click=”add(2, $event)”

定义事件时：add(n, e) { // xxx }

通过 e.target 就可以获取触发事件的对象，从而可以进行一系列操作，如修改css样式等

条件渲染指令

v-if

  • 会动态的创建或移除DOM元素，从而控制元素在页面上的显示与隐藏
  • 切换开销更高
  • 频繁切换就用v-show

v-show

  • 动态地为元素添加或移除 style=“dsipaly:none;” 样式，从而控制元素的显示与隐藏

  • 初始渲染开销更高

  • 运行时条件很少改变，就用v-if

循环渲染指令

v-for 使用的是 item in items 形式地特殊语法

v-for="(item, index) in list"其中item index可以根据需求进行重命名

官方建议：

  1. 只要用到了 v-for 指令，那么一定要绑定一个 :key 属性
  2. 而且，尽量把 id 作为 key 的值
  3. 官方对 key 的值类型，是有要求的：字符串或数字类型
  4. key 的值是千万不能重复的，否则会终端报错：Duplicate keys detected

使用v-for的时候需要绑定id，id可以利用nanoid自动生成
